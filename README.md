# RouteCert

Учебный full-stack MVP веб-сервиса для подбора и бронирования экскурсий с сертифицированными специалистами.

## Цель проекта

Разработать правдоподобный веб-сервис, в котором пользователь может найти экскурсию по реальному направлению, изучить профиль сертифицированного специалиста, проверить данные сертификата, рассчитать стоимость и создать демо-бронирование.

## Актуальность

Туристические сервисы всё чаще должны решать не только задачу поиска маршрута, но и задачу доверия к исполнителю. Для экскурсий важны квалификация гида, прозрачность сертификатов, понятная программа, отзывы, стоимость и удобное бронирование. Проект демонстрирует такую модель в формате, подходящем для курсовой работы.

## Стек

- **Next.js App Router + TypeScript** — страницы, server components и API routes в одном проекте.
- **React** — интерактивные фильтры, личный кабинет и форма бронирования.
- **Prisma ORM + SQLite** — локальная база данных без отдельного сервера.
- **Tailwind CSS** — адаптивный интерфейс и единый дизайн.
- **Zod** — валидация API-запросов бронирования.
- **OpenStreetMap embed** — отображение маршрута по координатам.
- **Open-Meteo API** — прогноз погоды без ключа, с mock fallback.

Такой стек выбран потому, что он выглядит как современный production-подход, но остаётся достаточно простым для запуска и объяснения в курсовой.

## Основные страницы

- `/` — главная страница с поиском, фильтрами, популярными экскурсиями, преимуществами и специалистами.
- `/catalog` — каталог экскурсий с фильтрацией и сортировкой.
- `/tours/[id]` — детальная страница экскурсии: программа, маршрут, координаты, карта, погода, отзывы, гид.
- `/guides/[id]` — профиль сертифицированного специалиста: опыт, языки, сертификат, отзывы, список экскурсий.
- `/booking` — форма бронирования с расчетом итоговой цены.
- `/account` — личный кабинет с профилем, активными бронированиями, отменой и историей.

## API endpoints

- `GET /api/tours` — список экскурсий с фильтрами: `q`, `country`, `city`, `maxPrice`, `minRating`, `maxDuration`, `category`, `language`, `certified`, `sort`.
- `GET /api/tours/:id` — одна экскурсия по `id` или `slug`.
- `GET /api/guides` — список специалистов.
- `GET /api/guides/:id` — профиль специалиста по `id` или `slug`.
- `POST /api/bookings` — создание бронирования.
- `GET /api/bookings` — бронирования демо-пользователя.
- `PATCH /api/bookings/:id` — изменение статуса, используется для отмены.
- `GET /api/reviews` — отзывы, можно фильтровать по `tourId` или `guideId`.
- `GET /api/cities` — список городов и регионов.
- `GET /api/certificate-check/:id` — mock-проверка сертификата специалиста.
- `GET /api/weather` — погода по координатам через Open-Meteo с fallback.
- `GET /api/price` — расчет стоимости бронирования.

## База данных

Prisma-схема находится в `prisma/schema.prisma`.

Основные модели:

- `User` — демо-пользователь.
- `Guide` — сертифицированный специалист.
- `Tour` — экскурсия с маршрутом, программой, координатами и привязкой к гиду.
- `Review` — отзывы по экскурсиям и специалистам.
- `Booking` — бронирования пользователя.
- `City` — направления и координаты.

Демо-данные находятся в `prisma/seed.ts`.

Скрипт `npm run db:push` генерирует Prisma Client и создаёт SQLite-таблицы через `prisma/create-db.ts`. Это делает запуск стабильным даже в окружениях, где бинарный `prisma db push` ограничен политиками выполнения.

В seed-файле создано:

- 21 экскурсия;
- 9 сертифицированных специалистов;
- 26 отзывов;
- 6 туристических направлений;
- 2 демо-бронирования для личного кабинета.

## Архитектура

```text
app/                 страницы Next.js и API routes
components/          переиспользуемые UI-компоненты
lib/                 Prisma client, сериализация, типы, расчет цены
prisma/              схема базы данных и seed-данные
public/              статические файлы
```

Данные читаются через Prisma на серверных страницах и через API в интерактивных client components. JSON-поля маршрута, программы и галереи хранятся в SQLite как строки и сериализуются в `lib/serializers.ts`.

- Создание бронирования.
- Личный кабинет с отменой активного бронирования.
- История завершенных экскурсий.
- Состояния загрузки, ошибок и пустых списков.
