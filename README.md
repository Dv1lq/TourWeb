# RouteCert

RouteCert — учебный full-stack MVP веб-сервиса для поиска и бронирования экскурсий с сертифицированными гидами. Проект показывает полный пользовательский сценарий: от подбора маршрута и просмотра профиля специалиста до расчета стоимости, проверки сертификата и создания демо-бронирования.

## Назначение проекта

Проект разработан как демонстрационный сервис для туристической платформы, где важны не только каталог и удобный поиск, но и доверие к исполнителю. В приложении есть реальные сущности предметной области: экскурсии, гиды, сертификаты, отзывы, города, маршруты, цены и бронирования.

RouteCert можно использовать как основу для курсовой, дипломного MVP, учебного full-stack проекта или прототипа сервиса бронирования.

## Основные возможности

- Поиск экскурсий по названию, городу, региону, стране и описанию.
- Фильтрация по стране, городу/региону, категории, языку, рейтингу, длительности, цене и статусу сертификации гида.
- Сортировка каталога по рейтингу, цене и длительности.
- Детальная страница экскурсии с программой, маршрутом, галереей, отзывами, погодой и профилем гида.
- Профили сертифицированных гидов с опытом, языками, рейтингом, номером сертификата и списком экскурсий.
- Отдельная страница всех сертифицированных специалистов.
- Mock API для проверки сертификата по `id`, `slug` или номеру сертификата.
- Бронирование экскурсии с клиентской формой, валидацией и расчетом итоговой цены.
- Личный кабинет демо-пользователя с активными и завершенными бронированиями.
- Отмена/изменение статуса бронирования через API.
- Погодный виджет через Open-Meteo с локальным fallback.
- Карта маршрута на Leaflet/OpenStreetMap.
- Локальная SQLite-база через Prisma.
- Проверка наличия изображений туров отдельным скриптом.

## Стек

### Frontend

- **Next.js 16 App Router** — страницы, layouts, server components и API routes в одном приложении.
- **React 19** — интерактивные клиентские компоненты: каталог, фильтры, бронирование, личный кабинет, карты.
- **TypeScript 5** — типизация компонентов, API и доменных моделей.
- **Tailwind CSS 3** — адаптивный интерфейс, дизайн-токены и переиспользуемые utility-классы.
- **Lucide React** — иконки интерфейса.
- **Leaflet + React Leaflet** — отображение маршрутов и точек экскурсий на карте.

### Backend и данные

- **Next.js Route Handlers** — REST-like API внутри `app/api`.
- **Prisma ORM 5** — схема данных, Prisma Client и доступ к SQLite.
- **SQLite** — локальная база `prisma/dev.db`.
- **Zod** — валидация входящих данных для бронирований и изменения статуса.
- **Open-Meteo API** — прогноз погоды без API-ключа.
- **date-fns** — работа с датами.

### Инструменты

- **ESLint 9 + eslint-config-next** — статическая проверка кода.
- **tsx** — запуск TypeScript-скриптов для seed и служебных проверок.
- **PostCSS + Autoprefixer** — обработка CSS.

## Быстрый запуск

Требуется Node.js и npm.

```bash
npm install
npm run db:push
npm run db:seed
npm run dev
```

После запуска приложение доступно по адресу:

```text
http://localhost:3000
```

Если в Windows PowerShell включена ограниченная execution policy, можно запускать npm через `npm.cmd`:

```bash
npm.cmd install
npm.cmd run db:push
npm.cmd run db:seed
npm.cmd run dev
```

## Переменные окружения

Пример находится в `.env.example`:

```env
DATABASE_URL="file:./dev.db"
NEXT_PUBLIC_APP_URL="http://localhost:3000"
```

`DATABASE_URL` используется Prisma для подключения к SQLite. При текущей конфигурации база лежит в директории `prisma`.

## API

| Метод и путь | Описание |
| --- | --- |
| `GET /api/tours` | Список экскурсий. Поддерживает `q`, `country`, `city`, `maxPrice`, `priceCurrency`, `minRating`, `maxDuration`, `category`, `language`, `certified`, `sort`. |
| `GET /api/tours/[id]` | Одна экскурсия по `id` или `slug`, вместе с гидом и отзывами. |
| `GET /api/guides` | Список гидов. Поддерживает `q`, `country`, `city`. |
| `GET /api/guides/[id]` | Профиль гида по `id` или `slug`, его экскурсии и отзывы. |
| `GET /api/cities` | Список городов. Поддерживает `q` и `country`. |
| `GET /api/reviews` | Список отзывов. Можно фильтровать по `tourId` или `guideId`. |
| `GET /api/price` | Расчет цены бронирования по `tourId`, `people`, `date`. |
| `GET /api/weather` | Погода по `city`, `lat`, `lon` через Open-Meteo с fallback-значениями. |
| `GET /api/certificate-check/[id]` | Mock-проверка сертификата по `id`, `slug` или `certificateNumber`. |
| `GET /api/bookings` | Бронирования пользователя. По умолчанию используется `demo-user`, можно передать `userId`. |
| `POST /api/bookings` | Создание бронирования. Тело валидируется через Zod. |
| `PATCH /api/bookings/[id]` | Обновление статуса бронирования: `confirmed`, `pending`, `completed`, `cancelled`. |

## Расчет стоимости

Логика находится в `lib/pricing.ts`.

Формула:

```text
subtotal = basePrice * people
serviceFee = subtotal * 5%
weekendSurcharge = subtotal * 12%, если дата выпадает на субботу или воскресенье
groupDiscount = subtotal * 8%, если участников 5 или больше
totalPrice = subtotal + serviceFee + weekendSurcharge - groupDiscount
```

Каталог также умеет фильтровать цены в разных валютах через `lib/currency.ts`. В проекте используются демо-курсы для `RUB`, `USD`, `EUR`, `CHF`, `JPY`, `CNY`, `AED`.

## База данных

Prisma-схема находится в `prisma/schema.prisma`.

Основные модели:

- `User` — демо-пользователь.
- `Guide` — сертифицированный гид.
- `Tour` — экскурсия с маршрутом, программой, галереей и связью с гидом.
- `Review` — отзыв по экскурсии и/или гиду.
- `Booking` — бронирование пользователя.
- `City` — город или направление с координатами.

JSON-поля маршрута, программы и галереи хранятся в SQLite как строки и сериализуются в `lib/serializers.ts`.

`npm run db:push` использует кастомный скрипт `prisma/create-db.ts`, который генерирует Prisma Client и создает таблицы SQLite. Это сделано для более стабильного запуска в окружениях, где прямой `prisma db push` может быть ограничен политиками выполнения.

## Особенности текущей реализации

- Проект является учебным MVP, поэтому полноценная регистрация и авторизация не подключены.
- Проверка сертификата реализована как mock API поверх данных гидов.
- Платежи не подключены, бронирование создается сразу в статусе `confirmed`.
- Open-Meteo не требует ключа, но при недоступности внешнего API приложение возвращает fallback-погоду.
- Карта использует OpenStreetMap/Leaflet и локальные координаты маршрута.
- SQLite подходит для локального демо, но для production стоит заменить БД на PostgreSQL или другой серверный вариант.
